Passo 1: primitive per il problema.
creare modelling_P_lib contenente:
    1. gen_instance(descrittori, seed)
        -> necessaria per generare le istanze.

        -> necessaria per controllare la glpsol solution.
    3. instance_to_str(instance, format)
        -> converte l'istanza in una stringa nel formato selezionato.
    4. get_instance_from_str(instance_str, format)
        -> data una stringa nel formato specificato, ritorna l'istanza.
    5. realizzare i convertitori per passare ai vari formati




Passo 2: generazione automatica istanze.
creare la cartella gen contenente:
    1. file GEN che contiene i comandi per la generazione automatizzata delle istanze.
        -> add_format: aggiunge un formato. Ogni istanza verrà generata nei formati specificati
        -> suite: aggiunge una cartella in instances. I goal di eval dovranno corrispondere alle varie suite disponibili.
        -> COPY: copia un istanza hardcoded nella suite corrente
        -> GEN: generata un istanza nella suite corrente
    2. instance_generator.py
        -> questo file deve leggere come argomenti i descrittori di istanza (come: m, n, seed, ...) e il format, per poi stampare l'istanza a video.
    3. il symlink a TAL_utils/problem_maker/TAL_GEN_instances 
        -> esso chiamerà instance_generator.py, passandogli gli argomenti letti dal file GEN, effettuando la generazione di tutte le istanze descritte in quel file)
    4. il symlink a services/modelling_P_lib
        -> esso verrà usato da instance_generator per generare l'istanza di P e stamparla
    5. in hardcoded/ è possibile inserire delle istanza che poi potranno essere copiate in instances/ mediante il comando COPY in GEN
